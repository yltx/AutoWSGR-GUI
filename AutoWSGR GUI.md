## AutoWSGR GUI

### MVP 核心功能

#### 初次配置

- 完成 AutoWSGR core 的安装（通常在首次运行时）
  - 独立安装配置，**不碰环境**，将 Python 安装到可控目录下并在线通过镜像安装依赖。
  - 安装正常完成后，在可控目录打安装完成的标记，下次检查到标记后直接跳过安装。
- 检查模拟器情况（可以把这些交给后端，直接 HTTP）
  - 查找注册表检查安装情况
  - 通过命令行、Adb 查看运行情况
    *Adb、Python 资源需要前端安装时打包携带。*

#### 导入战斗文件并执行

- 以图形界面描述性展示被导入的 yaml 战斗文件
  - 可以做一个 Page
- 运行单一战斗文件、固定每 15min 检查远征并尝试收取

#### 用户配置

**决战配置和执行**

- 配置 level1，level2，flagship
- 默认快修打完一轮

**日常调度**

- 自动演习调度（空闲时）（调度哪队）
- 自动战役调度（空闲时）（调度打哪个）

**模拟器设置**

- 可以结合自动检测功能，也可以手段选择

#### 页面设计

- 主页面：展示当前状态，展示 ongoing 的战斗，选择/导入 战斗方案，提供 "执行决战"，"执行当前任务" 等
- 导入预览页：导入 yaml 时提供一个预览，查看方案情况，ongoing 的战斗点 "查看策略" 也可以看这个策略
- 配置页面：可以直接做到主页面，配置 "用户配置"。

#### 其它

**导出成独立 .exe 安装文件**

- 双击运行自动解压有关文件到特定目录（默认桌面），并生成有关快捷方式图标。
  - 可以用有关的框架，也可直接写 cpp，附带有关资源并静态编译


### V1 拓展

#### 展示增强

- 挑选舰船、配置 Fleet 时展示舰船信息（利用图片），可以展示等级、血量状态等。等级可以一定支持手动修改（用于调度）

#### 练级调度

**依赖**：

- 文字换船功能完善
- FightResult 返回有关的等级信息
- 实现探查舰船当前等级的功能

**特性**：
- 提供一个任务模板，一些预设的练级模式（例如炸鱼模式、夜战模式、航空战模式）
- 用户指定一组需要练级的舰船（可能需要填入当前等级）
- 系统根据等级情况，破损情况，练级模式，自动安排合理的练级战斗方案和修复方案

#### 特殊、日周常调度

**依赖**：
- MissionPage 提供识别当前任务功能（基于 OCR 和数据库）

**特性**
- 提供一个任务模板，要求用户填写若干个预设编队（例如潜艇队、驱逐队、某些特定任务的自定义编队）
- 检索当前未完成的任务，使用这些预设编队自动安排合理的战斗方案和修复方案。
- 一般来说，完成各个任务所用的地图是固定的（暂时这么做）

#### 战利品调度

**特性**
- 提供一个任务模板，用户可以选择常见地图并配置有关编队
- 自动安排合理的战斗方案和修复方案，目标是获取特定的战利品
- 当日获取满时，自动停止
- 对用户展示：是否启用每日自动战利品

#### 决战调度

**依赖**：

- 决战实现中断功能
- 增强决战舰队配组功能（根据大破情况能配组出合理的舰队）
- 浴场修理（点名修理）功能恢复
- 剩余决战次数探测

**特性**：
- 提供决战任务配置模板（MVP 已经完成），可以选择是否启用快修
- 当无法组出合理队伍时应该调度其它任务并等待维修完成。
- 需要保存修理时间点等信息，以便在重启后继续执行。
- 对用户展示：是否启用自动刷盘（每天几张，最低留几张、用资源购买磁盘配置）；是否启用自动快修（默认启用）

#### 长时间挂机调度

- 定时器生产者
- 演习刷新时间自动生成演习任务，每日自动生成战役任务，每日任务、每周自动生成周常任务
- 数据应该被可持久化，以便重启后继续执行
- 对用户展示：启用自动演习（默认第一舰队用于演习）；启用自动战役（选择类型）；启用自动日常（选择做哪些任务）；启用自动周常（选择打到哪个图）；

#### 任务模板

- 由任务模板可以生成一个任务，任务可以被调度器调度。
- 提供配置项按照规则每日自动生成任务，提供一些便捷按钮一键生成一些任务
- 将导入 yaml 的功能也作为一个任务模板，用户可以选择导入一个 yaml 文件来生成一个任务
- 目前有的任务地图是写死的，需要考虑未来可配置。

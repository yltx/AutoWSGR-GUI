## 阶段一：基础框架与 MVP 核心功能

### 基础框架

- [x] Electron + TypeScript 项目搭建（esbuild 打包）
  说明：主进程使用 TypeScript，渲染进程通过 esbuild 打包，已具备开发、构建、打包发布链路。

- [x] MVC + ViewObject 架构
  说明：当前已拆分为 controller、model、view，并通过 ViewObject 向视图传递渲染数据。

- [x] ApiClient：HTTP + WebSocket 与 Python 后端通信
  说明：已支持 REST 调用、日志 WebSocket、任务状态 WebSocket、自动重连。

- [x] Scheduler 调度器：任务队列管理、顺序执行、状态回调
  说明：已支持任务队列、顺序执行、优先级、停止、清空、拖拽重排、完成回调。

- [x] 日志面板：前端本地日志 + 后端 WebSocket 实时日志
  说明：主页面已提供日志面板，能显示启动、环境检测、后端运行与调度日志。

- [x] 多任务类型支持：普通战斗、演习、战役、决战、活动、远征检查
  说明：任务模型和调度器均已支持多种任务类型。

- [x] 任务预设 YAML 导入：直接加入队列
  说明：支持直接导入 preset yaml 生成任务，无需先手动编辑完整方案。

### 初次配置

- [x] 完成 AutoWSGR core 安装（通常在首次运行时）
  说明：当前采用 Python 子进程 + pip 安装 autowsgr 的方案，首次运行会自动检查并安装依赖。

- [x] 独立安装配置，不碰系统环境，将 Python 安装到可控目录并在线安装依赖
  说明：当前使用本地 python 目录和 site-packages 目录，通过 pip install --target 安装，不污染全局环境。

- [ ] 安装完成后打安装标记，下次检查到标记直接跳过安装
  说明：当前没有单独的“安装完成标记文件”机制，仍然会在启动时进行环境检查。
  说明：v1.1.2 已优化为缓存 Python 路径并将依赖检查合并为单次调用，但仍属于“快速检查”，不是“完全跳过”。

- [x] 检查模拟器安装情况：查找注册表检查安装情况
  说明：已支持检测 MuMu、雷电、蓝叠的注册表项，并自动回填可执行路径与默认串口。

- [ ] 检查模拟器运行情况：通过命令行、ADB 查看运行情况
  说明：当前仅能检测“是否已安装”和路径，尚未检查“是否正在运行”“ADB 是否在线”。
  说明：后续需要后端或主进程补充 adb devices、连接测试、进程状态检测。

- [ ] ADB、Python 资源在前端安装时打包携带
  说明：Python 已打包并可自动准备；ADB 目前主要依赖模拟器自带 adb，可执行文件路径来自模拟器安装目录。
  说明：若要完全自携带 ADB，需要新增资源打包、版本管理与优先级策略。

  说明：目前任务组已可持久化，但“流程模板”概念尚未单独抽象与管理。

### 容错与重试

- [ ] 任务失败自动重试：可配置重试次数、间隔
  说明：当前任务失败后不会自动重试。
  说明：前端调度器可以实现重试策略，但需要后端提供更稳定的失败类型、可重试判定和幂等保障，避免重复执行造成副作用。

- [ ] 异常恢复策略：连接丢失重连、卡死检测重启
  说明：WebSocket 已有自动重连，但“系统卡死检测”“自动重启后端或任务恢复”尚未建立完整机制。
  说明：后端依赖：需要提供健康检查、当前任务可恢复状态、可中断安全点等信息。

- [ ] 任务跳过机制：连续失败 N 次后自动跳过当前任务
  说明：前端尚未记录连续失败次数与跳过策略。
  说明：后端依赖：需要可靠区分“暂时失败”“配置错误”“不可恢复失败”。

- [x] 日志级别过滤：debug / info / warning / error 分级显示
  说明：已实现过滤按钮栏（调试/信息/警告/错误）+ 计数badge + 切换显隐。
- [x] 日志美化：图标+文字联合说明模式
  说明：每条日志带级别SVG图标、彩色左边栏、频道pill标签、分层布局，截图友好。
